# Project Create 모드 AI 버튼 활성화

## Notes

### Summary
Project 생성(create) 모드에서 AI 버튼들이 비활성화되던 문제를 해결. entity_id 없이도 entity_name으로 AI 추론 가능하도록 백엔드/프론트엔드 수정.

### Changes Made
- **api/routers/ai.py**
  - `InferMultiDockRequest`, `InferHypothesisMapRequest`, `InferProjectSummaryRequest` 모델에 `entity_name`, `entity_description` 필드 추가
  - `entity_id`를 optional로 변경
  - `infer_multi_dock`, `infer_hypothesis_map`, `infer_project_summary` 엔드포인트에서 entity_id 없을 때 request 파라미터로 임시 entity 생성하는 로직 추가

- **dashboard-v2/src/features/ai/api.ts**
  - Request 타입에 `entity_name?`, `entity_description?` 추가
  - `entity_id`를 optional로 변경
  - API 함수에서 새 파라미터 전달

- **dashboard-v2/src/features/projects/components/ProjectForm.tsx**
  - AI 핸들러들에서 `formData.entity_name`, `formData.description` 사용
  - 버튼 disabled 조건: `!id` → `!id && !formData.entity_name`
  - 활성화된 버튼: multi-dock, hypothesis-map, project-summary
