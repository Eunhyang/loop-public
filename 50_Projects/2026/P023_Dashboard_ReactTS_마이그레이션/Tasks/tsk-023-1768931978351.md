# Schema - Track quarter_objectives 필드 추가

## Summary

Track 엔티티에 분기별 측정 가능한 목표를 구조화하여 저장하는 `quarter_objectives` 필드 추가.
기존 `quarters` (배열) 대신 맵 형태 `{YYYYQN: {...}}` 구조로 대시보드/롤업/자동화 지원.

## Changes Made

### 1. schema_constants.yaml (v5.9 → v5.10)
- `known_fields.Track`에 `quarter_objectives` 추가
- `field_structures.quarter_objectives` 구조 정의:
  - key_pattern: YYYYQN (예: 2026Q1)
  - fields: objective, kpi, target, owner?, evidence_link?

### 2. _template_track.md
- 템플릿에 `quarter_objectives` 예시 추가

### 3. 6개 Track 파일 업데이트
- Track_1_Product.md: Q1 Loop Mapping v0.6 → Q4 v1
- Track_2_Data.md: Q1 패턴 5개 → Q4 IRB 서류 완료
- Track_3_Content.md: Q1 키워드 10개 → Q4 콘텐츠 50개
- Track_4_Coaching.md: Q1 코치 4명 → Q4 라벨링 150건/주
- Track_5_Partnership.md: Q1 타겟 5곳 → Q4 케이스 10명
- Track_6_Revenue.md: Q1 월매출 1,200만 → Q4 런웨이 18개월

## 짧은 회고

기존 quarters(배열)와 별개로 맵 형태 quarter_objectives 추가.
분기별 lookup O(1), 연도 확장 용이, 롤업 자동화 가능.
