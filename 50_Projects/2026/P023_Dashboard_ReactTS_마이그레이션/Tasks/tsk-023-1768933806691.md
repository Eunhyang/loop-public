# API - SSOT frontmatter 생성 로직 중앙화

## 설명

API의 생성 로직(hypotheses.py, projects.py, tasks.py)에서 frontmatter를 하드코딩하는 SSOT 위반 문제 해결.

## Notes

### Summary
ssot_service.py에 frontmatter 빌더 함수를 추가하여 생성 로직 중앙화. 새 필드 추가 시 빌더 함수만 수정하면 됨.

### Changes Made
- `api/services/ssot_service.py`: frontmatter 빌더 함수 3개 추가
  - `build_hypothesis_frontmatter()`: Hypothesis frontmatter 생성
  - `build_project_frontmatter()`: Project frontmatter 생성
  - `build_task_frontmatter()`: Task frontmatter 생성
- `api/routers/hypotheses.py`: 하드코딩 제거, 빌더 함수 사용
- `api/routers/projects.py`: 하드코딩 제거, 빌더 함수 사용
- `api/routers/tasks.py`: 하드코딩 제거, 빌더 함수 사용
