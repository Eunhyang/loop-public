  \ 외워서 말할 것 (3개)\n\n1. **30초 피치(암기)**\n\n- 문제(누구의 어떤 고통) → 해결(어떻게 바꿈) → 근거(이미 관찰한\
  \ 증거 1개) → 다음(90일에 뭘 증명)\n\n2. **팀/본인 ‘Founder DNA’ 60초(암기)**\n\n- “왜 내가 이 문제를 끝까지\
  \ 파나?” + “어떻게 빠르게 배우고 바꾸나?”\\\n  Antler가 보는 핵심이 *창업자 특성/실행력* 쪽이라는 걸 공식 글에서도 반복해요.\n\
  \n3. **Evidence Pack 90초(메모 보고 말해도 됨)**\n\n- (고객/사용자) 대표 케이스 2\\~3개\n\n- (수치) 지표\
  \ 2\\~3개(정의 포함)\n\n- (학습) 최근 4주 동안 “뭘 했고→뭘 배웠고→뭘 바꿨는지” 3줄\n\n### 2) 인터뷰 30분 운영 시나리오(권장)\n\
  \n- **0–2분:** 30초 피치 + 60초 창업자/팀 소개 + “오늘 꼭 확인받고 싶은 1가지”\n\n- **2–20분:** Q&A (문제\
  \ 선명도 / 왜 지금 / 왜 너희 / 실행 증거 / 팀 역할)\n\n- **20–28분:** 반박 대비(경쟁·리스크·유닛이코노믹스 가설)\n\n\
  - **28–30분:** “Antler에서 얻고 싶은 것” + 다음 인터뷰/과제 확인\n\n### 3) 자주 나오는 질문 10개(답변 뼈대)\n\
  \n- 왜 이 문제? 왜 지금?\n\n- 고객은 누구고, 가장 아픈 순간은 언제?\n\n- 대체재/경쟁 대비 뭐가 다름?\n\n- 지금까지의 가장\
  \ 강한 “증거” 1\\~3개는?\n\n- 실패/반대 신호는 뭐였고, 어떻게 수정했나?\n\n- 앞으로 90일에 어떤 가설을 어떻게 검증할 건가?\n\
  \n- 수익모델 가설(누가 왜 돈을 내나)?\n\n- 공동창업자/팀 역할은 어떻게 나뉘나? 빈자리는?\n\n- 본인 강점/약점과 보완 방식은?\n\
  \n- Antler에서 무엇이 가장 필요하고, 어떻게 활용할 건가?\n\n## Founder 줌세션 내용 \n\n# 파트너 인터뷰 준비: 세션\
  \ 핵심 정리 (복붙용)\n\n---\n\n## 1) 이번 세션 핵심 요약 (면접에서 ‘우선순위’로 써먹기)\n\n파트너 인터뷰의 우선순위는 사실상\
  \ 이 순서:\n\n1. **Team**(Founder DNA + 협업/학습/유연성)\n2. **Traction**(검증의 증거, “필드에서의\
  \ 증거”)\n3. **Market**(크기 자체보다 “우리가 정의해나가는 시장/대체재”가 더 중요)\n\n특히 인상적인 포인트:\n\n- “마켓\
  \ 정의는 지원자가 틀릴 확률이 높다 → 그래서 ‘TAM 크게 말하기’는 가점이 거의 없음”\n- 대신 “우리가 무엇을 대체하는가(기존 행동/기존\
  \ 솔루션/기존 워크플로우)”가 훨씬 중요\n- 아이디어 고정은 리스크: “나는 이것만 하겠다”보다 문제 정의가 깊고, 학습으로 피벗 가능한 유연성이\
  \ 강한 신호\n- **VC는 ‘빠른 성장의 기울기(slope)**’에 투자 → 지금 숫자가 작아도 검증 속도/학습 속도가 보여야 함\n- 유료화는\
  \ 절실함/가치 검증의 리트머스: “돈 내라 했을 때 반응”을 본다\n\n---\n\n## 2) 파트너 인터뷰에서 “반드시 준비하면 좋은 3가지\
  \ 산출물” (세션 버전으로 재정의)\n\n세션 내용을 기준으로, “피치덱”보다 아래 3개가 더 먹혀.\n\n### 산출물 A) WHO–Problem\
  \ Deep Dive (1페이지)\n\n“누구”가 핵심이랬지. 이 1페이지는 아래 질문에 답해야 해:\n\n- 누구인가? (가장 절실한 1명/1세그먼트)\n\
  - 그 사람은 그 문제를 해결하려고 이미 어떤 노력을 하고 있나?\n- 노력 안 하면 정말로 아픈가?(비용/감정/관계/업무)\n- 대안 솔루션이\
  \ 있는데도 왜 아직 기회가 남아있나?\n- 우리가 뭘 주면 “박수치며 돈을 낼” 정도인가?\n\n면접에서 이 1페이지를 기반으로 질문이 들어오면\
  \ “시장”이 아니라 “사람”으로 대답이 시작돼서 흐름이 안정돼.\n\n### 산출물 B) Validation Log (2주\\~4주 단위 타임라인)\n\
  \n파트너가 말한 “validation 보고”는 이런 형태:\n\n- 고객 대화/관찰 수(숫자)\n- 제품 없어도 한 행동: 대기리스트/사전결제/LOI/유료\
  \ 테스트/유료 의향 테스트\n- 거절/반대 신호와 그에 따른 수정(=겸손+학습 증거)\n\n포맷 예시(한 줄씩):\n\n- Week n: (행동)\
  \ 누구를 만나 무엇을 관찰 → (학습) 뭐가 틀렸고 → (수정) 그래서 뭘 바꿈 → (결과) 반응 변화\n\n### 산출물 C) 90-Day\
  \ Proof Plan (가설 1\\~2개, 실험 3개)\n\n“앞으로 90일에 뭘 증명하나?”는 거의 필수 질문이야.\\\n여기서 중요한 건\
  \ TAM이 아니라 검증의 속도/정확도.\n\n- 핵심 가설 1\\~2개 (예: 결제, 유지, 효과 중 하나)\n- 가설마다 실험 3개 + 성공/실패\
  \ 기준(숫자)\n- 실패 시 **다음 브랜치(대체 방향**)까지 써두면 “유연성” 점수 올라감\n\n---\n\n## 3) 면접 답변을 “세션\
  \ 우선순위”에 맞게 리라이트하는 방식\n\n### (1) Market 질문이 나오면 “크기”로 가지 말고 “대체재”로 간다\n\n템플릿:\n\n\
  - “We’re not betting on TAM slides. The real question is: what do we replace in\
  \ daily life/workflow.”\n- “Today, they solve it with A(대체 행동/도구), but it fails\
  \ because B(구조적 이유).”\n- “So we define the market as C(우리가 만드는 카테고리/새 워크플로우).”\n\
  \n### (2) Traction이 약하면 “팀 트랙션(실행력)”으로 보완\n\n템플릿:\n\n- “Numbers are early, but what\
  \ we can prove is speed of learning.”\n- “In the last 4 weeks, we ran X experiments,\
  \ changed Y, and got Z signal.”\n- “That’s why we believe we can iterate to PMF\
  \ faster than others.”\n\n### (3) Team을 묻지 않아도, 팀을 계속 보여줘야 한다\n\n파트너가 말한 강한 신호(그대로\
  \ 쓰기):\n\n- 에너지 레벨\n- 지적/날카로운 피드백에 겸손하게 흡수\n- “내가 틀릴 수 있다 → 바로 바꿔서 해보자”의 유연함\n-\
  \ 극한 환경에서 견디는지(=리스크/불확실성에서의 태도)\n\n---\n\n## 4) 30분 파트너 인터뷰 운영 스크립트 (세션 내용 반영 버전)\n\
  \n### 0–2분\n\n- 30초 피치 (문제=누구/고통 → 해결=대체재 교체 → 증거 1개 → 90일)\n- “오늘 제가 꼭 확인받고 싶은\
  \ 건 (WHO 정의/결제 절실함/유료화 신호 중 1개) 입니다.”\n\n### 2–18분 (Q&A)\n\n- 질문이 Market으로 가면: TAM\
  \ 말고 대체재/Workflow로 전환\n- 질문이 Team으로 가면: 협업/학습/피봇 사례 1개를 꼭 넣기\n\n### 18–26분 (Validation\
  \ Deep Dive)\n\n- Validation Log 4줄 + 가장 강한 고객 케이스 2개\n- “이 신호가 틀리면 우리는 X로 방향을 바꾸겠다”\
  \ (유연성)\n\n### 26–30분\n\n- 90-Day Proof Plan 3개 실험\n- “Antler에서 제일 필요한 건 (코파운더 매칭/글로벌\
  \ 문제 프레이밍/후속투자 브릿지 등)이고, 이렇게 활용하겠다”\n\n---\n\n## 5) 바로 써먹는 “자주 나오는 질문” 답변 뼈대 10개\
  \ (세션 기준)\n\n 1. **Why this problem / why now?**\n\n    - “이 문제는 (WHO)에게 (지속 비용)이라\
  \ 회피가 안 된다” + “지금은 (행동 변화/결제/유통) 레버가 생겼다”\n\n 2. **Who exactly?**\n\n    - “가장 절실한\
  \ 1 세그먼트” + “그들이 이미 하고 있는 노력 3가지” + “그래도 실패하는 이유 1개”\n\n 3. **What do you replace?**\n\
  \n    - “우리가 대체하는 것은 앱/코칭이 아니라 ‘기존의 해결 방식 A’다”\n\n 4. **Why you?**\n\n    - “도메인에서\
  \ 깊게 박힌 문제정의” + “학습 속도(최근 4주 실험)” + “실행/유료화 의지”\n\n 5. **Strongest evidence?**\n\
  \n    - “현장 증거(대화/관찰)” + “유료화 반응/사전결제/LOI” + “행동 변화/유지 신호”\n\n 6. **What went wrong\
  \ and what changed?**\n\n    - “반대 신호 1개를 먼저 말하고” → “그래서 구조를 어떻게 바꿨는지”\n\n 7. **Revenue\
  \ model?**\n\n    - “누가 왜 돈을 내는지”를 WHO의 절실함으로 다시 연결\n\n 8. **Competition?**\n\n\
  \    - “경쟁사는 기능이 아니라 대체재(현재의 해결 방식)”로 정의\n\n 9. **90 days?**\n\n    - “가설 1\\~2개”\
  \ + “실험 3개” + “성공/실패 기준” + “실패 시 브랜치”\n\n10. **Why Antler?**\n\n- “코파운더/시장 정의/후속투자”\
  \ 중 1\\~2개로 구체화\n- \n\n---\n\n## 작업 내용\n\n면접:\n\n<table class=\"tiptap-table\" style=\"\
