# sosi - 식전 사진 카메라 전면→후면 버그 수정

## 설명

식사 타이머 시작 시 "식전 사진 찍기" 기능을 ON하면 카메라가 **전면(셀프) 카메라**로 열리는 버그.
음식 사진을 찍어야 하므로 **후면(rear) 카메라**로 열려야 함.

### 원인
`CameraDevice.front` → `CameraDevice.rear`로 변경 필요

## 핵심 파일

| # | 파일 | 수정 위치 |
|---|------|---------|
| 1 | `lib/presentation/timer/widgets/pre_meal_camera_modal.dart` | 52행 |
| 2 | `lib/data/services/timer/timer_launch_service.dart` | 139행, 320행 |

## 데이터 흐름

```
사용자: "식전 사진 찍기" 클릭
    ↓
PreMealPhotoModal._onTakePhotoPressed()
    ↓
TimerLaunchService.capturePreMealPhoto() [320행]
    ↓
ImagePicker.pickImage(preferredCameraDevice: CameraDevice.front) ← 버그
    ↓
전면(셀프) 카메라 열림
```

## 체크리스트

- [ ] timer_launch_service.dart:320 - CameraDevice.front → rear
- [ ] 실기기 테스트
