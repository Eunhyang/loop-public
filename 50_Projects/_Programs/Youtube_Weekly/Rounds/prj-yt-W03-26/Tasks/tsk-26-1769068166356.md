notes: "# LOOP 콘텐츠용 UI MVP 기획안 (Alarmy형 “통과 조건”)\n\n목표: **“패턴이 시작되기 전에, 아주 짧은 ‘패스’를\
  \ 통과한다**”를 한 컷에 이해시키는 UI를 만든다.\\\n범위: **실제 기능 구현 X**, 콘텐츠(영상)에서 보여줄 **UI 프로토타입/목업**만\
  \ 제작.\n\n---\n\n## 0) 톤 가이드 (공통)\n\n- 금지: “폭식”, “위기”, “위험”, “통제”, “실패”\n\n- 권장:\
  \ “패턴”, “자동”, “스위치”, “리셋”, “잠깐 멈춤”, “오늘도 충분히 잘했다”\n\n- 기본 문장 공식: **“지금” + “한 번만”\
  \ + “10초/30초/1분/10분”**\n\n- ‘강제’ 느낌 금지: 사용자는 항상 **선택권**이 있다. (Secondary 버튼 제공)\n\
  \n---\n\n## 1) MVP 구성 요약 (UI 3장 + 케이스 2개)\n\n### 공통 UI 3장\n\n1. **Push 카드(진입 1탭)**\n\
  \n2. **PASS(통과 조건) 화면**: 링 타이머 + 미션 1개 + “완료”\n\n3. **PASS 완료 화면**: 보상(차분/선택권 회복)\
  \ + 다음 선택 2개\n\n### 케이스 2개\n\n- **이동(퇴근길)**: “30초 PASS” + **조용한 미션(호흡/감각)**\n\n\
  - **집(밤)**: “1분 PASS” + **행동 미션(물/세수/샤워 시작)**\n\n> 제작 효율: 2편을 찍어도 UI는 동일. **미션 카드\
  \ + 푸시 문구만 교체**.\n\n---\n\n## 2) 사용자 플로우 (콘텐츠용)\n\n### Flow A: 이동(지하철/버스)\n\n- (장면)\
  \ 배달 앱 켜려는 손\n\n- (푸시) “퇴근길 자동 스위치…” → 탭\n\n- (PASS 30초) 조용한 미션 수행\n\n- (완료) “좋아요.\
  \ 자동 대신 선택” + 2버튼\n\n### Flow B: 집(소파/현관)\n\n- (장면) 집 도착/소파 앉으며 습관 시작\n\n- (푸시)\
  \ “지금 자동 스위치…” → 탭\n\n- (PASS 1분) 물 한 컵(또는 세수 시작)\n\n- (완료) 동일\n\n---\n\n## 3) 화면\
  \ 상세 스펙 (바로 구현용)\n\n### Screen 1) Push 카드 (OS 스타일)\n\n**목적:** “지금 그 시간” + “한 번만\
  \ PASS”를 1초 내 이해.\n\n**공통 레이아웃**\n\n- 앱 아이콘 + 앱명 “LOOP”\n\n- Title 1줄 + Body 1줄\n\
  \n- 액션 버튼 1개(Primary)\n\n**A. 이동 푸시 카피**\n\n- Title: `퇴근길, 자동 스위치 켜지기 쉬운 구간`\n\n\
  - Body: `30초만 리셋하고 갈까요?`\n\n- Button: `30초 리셋`\n\n**B. 집 푸시 카피**\n\n- Title: `지금,\
  \ 자동 스위치 켜지기 쉬운 순간`\n\n- Body: `한 번만 1분 리셋해요.`\n\n- Button: `1분 리셋`\n\n**애니메이션/연출(콘텐츠용)**\n\
  \n- 푸시 등장: 기본 iOS/Android 스타일\n\n- 탭 시: 즉시 Screen 2로 전환 (0.2s)\n\n**이벤트(가짜로라도 UI에\
  \ 반영)**\n\n- push_opened\n\n- pass_started_from_push\n\n---\n\n### Screen 2) Reset\
  \ (통과 조건) — 히어로 화면 (가장 강렬)\n\n**목적:** Alarmy의 “미션 통과 전엔 끝나지 않음”을 우리 식으로 각인.\n\n\
  **핵심 컨셉**\n\n- ‘알람’처럼 시끄럽게가 아니라, **“Reset 링”이 올라오며 스위치를 바꾸는 장면**\n\n- 입력/판단 최소.\
  \ **미션은 1개**.\n\n- **완료하면 “Reset 완료”가 명확하게 보임(체크/진동/색 변화)**\n\n**레이아웃 (공통)**\n\n\
  - 상단 헤더(센터 정렬)\n\n  - 헤드라인: `지금은 ‘무의식 패턴’이 먼저 나오는 순간`\n\n  - 서브: `한 번만, {30초/1분}\
  \ 리셋해요`\n\n- 중앙: **원형 링 타이머**\n\n  - 링 안 텍스트(큰 글씨): `{30초 `**Reset**`}` 또는 `{1분\
  \ `**Reset**`}`\n\n  - 링 하단 작은 텍스트: `지금은 ‘결정’이 아니라 ‘리셋’`\n\n- 하단: **미션 카드 1개**\n\
  \n  - 카드 타이틀: `오늘의 `**Reset**` 미션`\n\n  - 카드 내용: 미션 문구 1줄\n\n  - 상태 버튼: `시작` → 진행중\
  \ → `완료했어요`\n\n- 최하단(작게): `여기까지만 해도 충분해요.`\n\n**A. 이동 미션(30초) 옵션 (1개만 채택)**\n\n\
  - 추천 1순위(티 안 남, 양손 X):\n\n  - 미션 문구: `1분 간단 알아차림`\n\n  - 버튼: `시작` 누르면 링이 1분 동안 자동\
  \ 진행\n\n  - 진행 중: 6초 템포마다 **햅틱(약하게) 1번** + “내쉬기” 텍스트만 깜빡\n\n  - 완료: 자동으로 “완료 ✅”\
  \ 전환\n\n- 대안(감각 앵커링):\n\n  - `손잡이/의자 감각 3개만 느끼기` (30초 타이머)\n\n**B. 집 미션(1분) 옵션 (1개만\
  \ 채택)**\n\n- 추천 1순위(성공률 최상):\n\n  - `물 한 컵`\n\n  - 버튼: `마셨어요`\n\n- 대안(환경 전환 강함):\n\
  \n  - `샤워기 물틀기 시작`\n\n  - 버튼: `했어요`\n\n**상태(STATE) 정의**\n\n- idle: 링 정지, 버튼 “시작”\n\
  \n- running: 링 진행(카운트다운), 버튼 비활성 또는 “진행 중…”\n\n- completed: 링 100% + 체크 ✅ + “PASS\
  \ 완료” 라벨 표시\n\n**완료 순간(보상 연출)**\n\n- 완료 시:\n\n  - 중앙에 체크 ✅ 0.4s 팝업\n\n  - 햅틱 1회(성공)\n\
  \n  - 하단 문구 1줄 등장: `좋아요. 자동 스위치가 잠깐 꺼졌어요.`\n\n- 0.6s 후 Screen 3로 자동 전환(또는 버튼 “다음”)\n\
  \n**이벤트**\n\n- pass_mission_started\n\n- pass_mission_completed\n\n- pass_mission_skipped(옵션:\
  \ “나중에” 있을 경우만)\n\n---\n\n### Screen 3) Reset 완료 — 보상 + 다음 선택(2갈래)\n\n**목적:** “각성”이\
  \ 아니라 **차분함/선택권 회복**을 보상으로 만든다.\n\n**레이아웃 (공통)**\n\n- 상단 타이틀: `좋아요. 무의식에서 ‘선택’이\
  \ 된 순간이에요.`\n\n- 서브: `오늘은 ‘시작 버튼’ 전에 한 번 멈췄어요.`\n\n- (초간단 보상 선택 칩 1개) — *갈망/배고픔\
  \ 없이도 가능*\n\n  - 질문: `지금 가장 가까운 느낌은?`\n\n  - 칩(4개): `조금 가벼움 / 조금 편안함 / 그대로 / 모르겠음`\n\
  \n- 버튼 2개\n\n  - Primary: `10분만 리셋하고 결정`\n\n  - Secondary: `오늘은 여기까지`\n\n- 하단 마이크로카피:\
  \ `완벽이 아니라 반복이 바꿔요.`\n\n**옵션(갈망/배고픔 UI는 “선택”으로만)**\n\n- “자세히 보기(선택)” 링크 1줄\n\n \
  \ - 누르면 모달로 슬라이더 2개:\n\n    - `먹고 싶은 마음` 0\\~10\n\n    - `몸의 배고픔` 0\\~10\n\n  -\
  \ 모달 닫기: `저장` / `닫기`\n\n> 콘텐츠 MVP에서는 모달은 굳이 안 보여줘도 됨. (복잡도↑)\n\n**이벤트**\n\n- pass_completed_screen_viewed\n\
  \n- feeling_chip_selected\n\n- choose_reset_10min / choose_done_for_today\n\n---\n\
  \n## 4) 컴포넌트 스펙 (디자이너/개발자 바로 쓰는 수준)\n\n### 4-1. 타이머 링\n\n- 길이: 이동 30초 / 집 60초\n\n\
  - 표시: mm:ss (예: 0:30 → 0:00)\n\n- 진행 애니메이션: linear\n\n- 완료 애니메이션: 체크 ✅ + 링 채움 고정\n\
  \n### 4-2. 미션 카드\n\n- 구성: 타이틀 / 1줄 설명 / 버튼\n\n- 버튼 상태:\n\n  - idle: “시작” (또는 “마셨어요/했어요”)\n\
  \n  - running: 비활성 + “진행 중…”\n\n  - completed: “완료 ✅” 고정\n\n### 4-3. 칩(보상 선택)\n\n\
  - 4개 중 1개 선택\n\n- 선택 후 즉시 “다음 선택 버튼” 강조(시각적으로)\n\n### 4-4. 버튼\n\n- Primary 1개, Secondary\
  \ 1개 고정\n\n- Secondary는 ‘도망’이 아니라 **존중**의 톤: `오늘은 여기까지`\n\n### 4-5. 햅틱/사운드\n\n-\
  \ 이동: 사운드 OFF 기본, 햅틱 약하게\n\n- 집: 사운드 없이도 충분. (콘텐츠에서는 효과음 추가 가능)\n\n---\n\n## 5)\
  \ 콘텐츠 촬영용 스토리보드(각 15초)\n\n### A편: 퇴근길(지하철) — “30초 Reset”\n\n1. (0–3s) 지하철, 폰으로 배달\
  \ 앱 열려는 손 + 자막 `퇴근길, 자동이 켜질 때`\n\n2. (3–5s) 푸시 등장 `30초 패스`\n\n3. (5–11s) Reset 링\
  \ 30초 + `숨 내쉬기 6초 × 3` (햅틱 느낌 연출)\n\n4. (11–15s) 완료 화면 `자동 대신 선택` + 버튼 2개\n\n###\
  \ B편: 집 — “1분 Reset”\n\n1. (0–3s) 집 도착/소파 앉는 장면 + 자막 `집에 오면 자동이 시작될 때`\n\n2. (3–5s)\
  \ 푸시 `1분 패스`\n\n3. (5–11s) Reset 링 1분 + 미션 `물 한 컵` + `마셨어요`\n\n4. (11–15s) 완료 화면\
  \ 동일\n\n---\n\n## 6) 제작 범위 체크리스트 (UI만)\n\n- Push 카드 2종(이동/집) — 카피만 다름\n\n- Reset\
  \ 화면 1종(공통) + 미션 카드 텍스트 2종\n\n- Reset 완료 화면 1종(공통)\n\n- 링 타이머 애니메이션(30초/60초) 2개\
  \ 프리셋\n\n- 체크 ✅ 완료 애니메이션 1개\n\n---\n\n## 7) 화면별 “한 줄 핵심 메시지”(콘텐츠 설명용)\n\n- Push:\
  \ “지금 ‘그 시간’에 먼저 톡 친다”\n\n- Reset: “패턴 시작 전에, 아주 짧은 통과 조건”\n\n- 완료: “차분해짐 + 선택권\
  \ 회복(그리고 다음은 내 선택)”"
