# Today(기록) 페이지 — 다이어리형 상태 기록 UX 설계안

## 전제
- Today 페이지는 "행동 입력 페이지"가 아니라
  → 하루를 쌓아두는 다이어리 페이지다
- + 버튼은 보조 수단
- 주요 기록은 페이지 상단에서 "항상 보이는 상태 입력"으로 이뤄진다

---

## 전체 구조 (현재 UI 기준)

[ 달력 (날짜 선택) ]

[ 상태 기록 바 (상단 고정) ]   ← 신규
--------------------------------
[ 식사 일기 리스트 ]
--------------------------------
[ + 버튼 (보조 입력) ]

---

## 상태 기록 바 (Core UI)

### 위치
- 달력 바로 아래
- 식사 일기 리스트 바로 위 
- 스크롤해도 고정 or 반고정

---

## 상태 아이콘 구성 (비활성 기본)

초기 상태:
- 모든 아이콘은 투명 / 비활성
- "기록 안 함"이라는 의미는 아님
- 그냥 아직 채워지지 않은 오늘의 페이지

💧  물   |   😖 스트레스   |   🌸 PMS
(아이콘 + 아주 작은 라벨)

---

## 인터랙션 설계 (핵심)

### 1️⃣ 아이콘 탭 → 즉시 슬라이드 입력

- 아이콘을 누르면
- 해당 아이콘만 활성화
- 페이지 이동 ❌
- 모달 ❌
- Bottom Sheet ❌

→ 아이콘 아래에 바로 슬라이더 등장

---

### 💧 물 기록 예시

UI:
- 0L —— 0.5L —— 1L —— 1.5L —— 2L+
- 손가락으로 쓸어 올리듯 조절

특징:
- 숫자 직접 입력 ❌
- 정확도 강요 ❌
- "대충 여기쯤" 감각적 입력

저장:
- 슬라이드 놓는 순간 자동 저장
- 완료 버튼 없음

---

### 😖 스트레스 기록 예시

UI:
- 게이지형 슬라이더
- 라벨 없음 또는 아주 추상적

낮음 —— 중간 —— 높음

의미:
- 감정 분석 ❌
- 오늘 전반적인 부담감 체감 정도

---

### 🌸 PMS 기록 예시

UI:
- 단계 선택형 or 짧은 슬라이드

Options:
- 해당 없음
- PMS 초기
- PMS 중
- PMS 후반

※ 날짜 계산 ❌
※ 주기 예측 ❌
※ 그냥 "오늘 느낌"

---

## 입력 완료 후 상태 변화

- 아이콘: opacity 100%
- 슬라이더 값은 접힘
- 아이콘 자체가 "오늘의 기록 결과"를 시각적으로 표현

예:
- 💧 아이콘 아래 작은 바 70% 채워짐
- 😖 아이콘 색 진하기로 강도 표현
- 🌸 아이콘 활성화 여부만 표시

---

## UX적으로 중요한 포인트 (절대 규칙)

### ❌ 하지 말 것
- 입력 완료 버튼
- “오늘 물 섭취량”
- “스트레스 관리 필요”
- 기록 알림 / 미입력 알림

### ⭕ 반드시 지킬 것
- 언제든 다시 눌러서 수정 가능
- 기록 안 해도 아무 말도 안 함
- 판단/피드백은 Today 페이지에서 절대 안 함

---

## + 버튼과의 관계 정리

### + 버튼의 역할
- 상세 식사 일기
- 사진 / 메모 / 식후감정

### 상태 기록 바의 역할
- 하루의 배경
- 식사를 해석하기 위한 맥락

→ 서로 경쟁하지 않고 역할이 다름

---

## 데이터 구조 관점 요약

- 상태 기록 바 입력
  → LoopStateWindow (daily / meso)
- 식사 일기
  → Event (meal)
- 두 데이터는
  → 같은 날짜 / 같은 dayKey로 자동 연결

---

## 이 설계의 가장 큰 장점

- 유저 입장:
  "오늘 페이지를 채워두는 느낌"
- 시스템 입장:
  "식사를 설명하는 고품질 맥락 데이터"
