notes: "### 1) Title / One-liner\n\n- 한 문장: “우리는 \\_*\\_ 문제를 \\_*\\_ 방식으로 해결한다”\n\n\
  - 보조 1줄: “현재까지의 증거: (숫자 2개만)”\n\n- 여기서부터 “말”이 아니라 “증거”의 톤을 잡아줌\n\n### 2) Origin\
  \ Story: 왜 시작했나 (필연)\n\n- 감정선은 짧고 날카롭게 (10\\~15초 읽히는 분량)\n\n- “이 문제를 *남의 일*로 생각했는데,\
  \ *우리 문제*였다”\n\n- **여기서 ‘진짜 질문’ 한 줄**: “왜 이런 문제는 해결 언어/도구가 없지?”\n\n### 3) Problem:\
  \ 이 문제가 얼마나 크고/아프고/반복되나\n\n- 시장 크기 말고, **반복 구조(루프**)를 보여줘요\n\n- “의지 → 실패 → 자책 →\
  \ 폭식/리바운드” 같은 **인과고리 1개**를 그림으로\n\n- 포인트: “이건 다이어트가 아니라, 정서-행동 시스템 문제다”\n\n### 4)\
  \ Insight: 기존 해결이 왜 실패했는가\n\n- 여기서 Antler이 좋아하는게 나옵니다: “그래서 우리는 가설을 이렇게 세웠다”\n\n\
  - 예시 형태:\n\n  - 가설1: “폭식은 음식 문제가 아니라 트리거-반응-보상 루프다”\n\n  - 가설2: “루프는 ‘측정/피드백/개입’이\
  \ 있어야 바뀐다”\n\n- **이 슬라이드는 ‘똑똑함’이 아니라 ‘문제 이해력’ 증명**\n\n### 5) Solution: 우리가 만든 ‘루프\
  \ 재설계’ 시스템\n\n- 기능 나열 금지. **메커니즘 중심**\n\n- “측정 → 해석 → 개입 → 회복” 4단계로\n\n- 코칭/앱이 함께\
  \ 있다면 “왜 둘 다 필요한지”를 딱 한 문장으로:\n\n  - “코칭은 깊이를 만들고, 앱은 확장을 만든다”\n\n### 6) Proof:\
  \ 행동이 먼저였고, 지표가 따라왔다\n\n- 이 슬라이드는 덱에서 제일 중요.\n\n- 원칙: **숫자 4개까지만, 전부 ‘행동-결과’로 연결**\n\
  \n  - “우리가 한 행동 A → 결과 지표 B”\n\n- 예시 포맷:\n\n  - “콘텐츠/커뮤니티로 수요 확인 → 가입자/활성/리텐션”\n\
  \n  - “코칭으로 메커니즘 검증 → 결제/완주/변화”\n\n- 그래프는 1개만. 나머지는 큰 숫자 카드.\n\n### 7) Learning\
  \ Loop: 가설–실험–학습–다음 액션\n\n- Antler가 좋아하는 “founder execution”을 여기서 보여줍니다.\n\n- 3줄\
  \ 템플릿으로만:\n\n  - 가설: \\__\\_\n\n  - 실험: \\__\\_ (2주 단위)\n\n  - 학습: \\__\\_\n\n \
  \ - 다음: \\__\\_ (다음 2주 액션)\n\n- 이거 한 장이면 “우리는 운이 아니라 방법으로 성장한다”가 증명돼요.\n\n### 8)\
  \ Why Us: 왜 우리가 이걸 제일 잘하나\n\n- 팀 소개 나열 금지. **권한(Authority)과 집착(Obsession)을 증명**\n\
  \n- 3가지 중 2개만 강하게:\n\n  1. 문제를 15년 겪은 당사자/가족으로서의 “집착”\n\n  2. 현장(코칭/트레이닝)에서 반복 패턴을\
  \ 본 “임상적 감각”\n\n  3. 데이터/제품으로 구조화해본 “시스템화 능력”\n\n- “우리는 이 문제를 ‘의지’로 보지 않는다” 같은 철학\
  \ 1줄이면 충분\n\n### 9) 12주 계획: Antler 안에서 뭘로 승부 볼 건가\n\n- ‘해야 할 일’ 나열 말고 **검증 플랜**\n\
  \n- 목표는 3개 KPI로만:\n\n  - Activation(첫 주 핵심행동)\n\n  - Retention(D30 같은)\n\n  - Paid\
  \ conversion(Trial→Paid)\n\n- “우리가 증명할 단 하나: \\__\\_가 반복 가능하게 만든다”\n\n### 10) Ask:\
  \ 우리가 원하는 것\n\n- Antler에서는 보통:\n\n  - “우리는 Antler에서 팀을 더 단단히 하고(또는 더 빠르게), \\__\\\
  _를 검증해 투자심사로 간다”\n\n- 자금/밸류에이션 얘기 최소화.\n\n## “넣지 말아야 할 것” 체크리스트\n\n- 시장 규모(TAM/SAM/SOM)\
  \ 과하게\n\n- 기능 목록/스크린샷 잔뜩\n\n- 경쟁사 표(특히 억지 차별점)\n\n- 장황한 연혁(한 장으로 끝)\n\n---\n\n##\
  \ 톤 가이드(지금 네가 말한 방향 그대로)\n\n- **스토리(필연) 20% + 검증(지표/실험) 60% + 비전(확장) 20%**\n\n-\
  \ “우리는 이렇게 믿는다”보다\\\n  **“우리는 이렇게 해봤고, 이렇게 바뀌었다”** 문장으로 통일"
