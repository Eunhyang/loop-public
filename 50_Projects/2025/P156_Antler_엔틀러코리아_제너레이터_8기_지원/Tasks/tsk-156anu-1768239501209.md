notes: "1. **귀하의 아이디어는 어떤 문제를, 어떤 방식으로 해결하나요? 아이디어를 간단히 설명해주세요.**\n\n**기존의 기록 중심 다이어트는\
  \ 폭식–자책으로 이어지는 정서 기반 반복 패턴을 끊어내기 어렵습니다. 저희는 반복적인 다이어트 실패로 섭식 통제력을 상실한 20\\~40대 여성을\
  \ 타깃으로, 셀프 관리층은 앱, 고위험·고관여층은 12주 코칭으로 접근합니다. 기존 앱이 수집하는 ‘무엇을 먹었는지(What)’를 넘어, 스트레스·정서·상황에서\
  \ 발생하는 ‘무너짐 순간(야식/폭식)’의 맥락과 즉시 실행 가능한 회복 개입, 그리고 결과를 표준 이벤트로 축적하는 시스템을 설계하고 있습니다.\
  \ 코칭에서 검증된 개입이 함께 저장되기 때문에 ‘무너짐→회복’에 대한 학습 가능한 데이터셋이 됩니다. 현재 ‘온전히 먹는 회복 루틴’만으로도\
  \ 앱 누적 사용자 6,700명, D1 50.4% / D30 20.5%의 사용 유지 신호를 확인했습니다.**\n\n2. **해당 아이디어의 비즈니스\
  \ 모델은 무엇인가요?**\n\n**저희는 코칭으로 ‘검증/데이터’를 만들고, 앱 구독으로 ‘스케일’을 만드는 2-트랙 모델로 수익화합니다. 12주\
  \ 1:1 코칭(꼭꼭FIT)에서 고관여 고객의 지불의사와 행동변화 조건(트리거·대체행동·루틴)을 검증하고, 그 과정에서 고품질 개입(회복 레시피)\
  \ 데이터를 축적합니다. 앱(꼭꼭)은 ‘무너짐 이후 복귀’에 필요한 핵심 루틴(타이머/포만감·감정 기록·회복 플로우)을 구독으로 제공해 반복 매출을\
  \ 만듭니다. 특히 GLP-1 확산으로 ‘감량’은 쉬워지는 반면, 약 중단 이후의 유지·리바운드·정서적 보상 욕구를 관리할 ‘회복 루틴’ 수요가\
  \ 커지고 있어 글로벌 확장 가능성이 높습니다.** \n\n**또한 유튜브/콘텐츠 기반 유기적 유입으로 마케팅 비용 의존도를 낮추고, 코칭을 통해\
  \ 고관여 고객의 지불의사를 확인했습니다. 단기적으로는 회복 개입 MVP 출시 후 기존 무료 영역까지 포함한 전면 유료화를 실행해 Trial-to-Paid와\
  \ 유료 D30 유지로 유료화 재현성을 검증하겠습니다. 장기적으로는 코칭에서 검증된 개입을 ‘무너짐→트리거→개입→결과’ 표준 이벤트로 축적해 개인화된\
  \ 개입 추천(언제/누구에게/무엇을)을 고도화하고, 같은 비용으로 더 많은 케이스를 커버하는 구조로 LTV를 키웁니다.**\n\n3. **이 아이디어로\
  \ 사업을 구축하기에 본인이 적합하다고 생각하는 이유는 무엇인가요?**\n\n**저희는 ‘섭식 문제의 회복(복귀)’을 당사자 인사이트 + 데이터/제품화\
  \ 역량 + 현장 운영 역량으로 끝까지 풀 수 있는 팀입니다. 직접 15년간 폭식·요요를 겪으며 문제의 본질이 ‘의지’가 아니라 ‘자책→절제→재폭식’이\
  \ 반복되는 패턴임을 체감했고, 공동창업자는 코칭 현장에서 고객의 트리거·대체행동·루틴을 매일 다루며 “행동이 바뀌는 조건”을 축적해왔습니다.\
  \ 또한 7년+ 백엔드/데이터 엔지니어링 경험을 바탕으로 코칭에서 검증된 회복 개입을 ‘무너짐→트리거→개입→결과’로 표준화해 데이터 자산으로 쌓고,\
  \ 이를 앱 기능과 실험으로 빠르게 전환하는 구조를 설계·구현할 수 있습니다. 실제로 콘텐츠로 유입을 만들고 코칭으로 유료 전환과 결과를 검증한\
  \ 뒤, 그 학습을 앱 구독으로 확장하는 실험을 반복해왔습니다. 결과적으로 저희는 “메시지→전환→유지→학습”이 연결된 데이터 선순환 성장 구조를\
  \ 굴릴 수 있는 팀입니다.**\n\n4. **앤틀러에 지원하게 된 동기와, 레지던시 기간 동안 이루고 싶은 목표를 알려주세요.**\n\n\n\
  1) /mcp-server rebuild\n\n## \n\n## **추천 전략: “역할 분리”로 중복 제거**\n\n### **1번(문제/해결)\
  \ =** \n\n### **한 문단 요약(앵커)**\n\n- 목적: *이 팀이 뭘 하는지 10초 안에 이해시키기*\n\n- 구성(4\\~6문장\
  \ 권장)\n\n  1. 타깃 + 핵심 문제(무너짐 이후 회복 공백)\n\n  2. 왜 기존 대안이 실패하는지(기록/의지 중심)\n\n  3.\
  \ 해결 메커니즘(복귀 순간 개입 + 루틴 재가동)\n\n  4. (선택) 증거 1줄(숫자 1문장만)\n\n여기에서 이미 **“정의/메커니즘”을\
  \ 끝내**두면, 2\\~4번은 “설계/수익/팀/실험계획”으로 자연스럽게 깊어져.\n\n### **2번(BM) =** \n\n### **현재 수익\
  \ + 확장 경로**\n\n- 목적: *돈 되는 구조인지 / 확장성 있는지*\n\n- 구성(결론 1줄 + 3\\~5 bullets)\n\n  -\
  \ 현재 BM: 앱 구독 + 12주 코칭 매출\n\n  - 왜 이 조합이 맞는지(코칭=검증/데이터축적, 앱=스케일)\n\n  - 단기 실험지표(Trial→Paid,\
  \ 유료 D30 등)\n\n  - 중장기 확장(개입 추천/자동화로 LTV↑, CAC↓ 방향)\n\n### **3번(왜 적합) =** \n\n###\
  \ **Founder–market fit 증명**\n\n- 목적: *이 팀이 이 문제를 끝까지 풀 수 있는 이유*\n\n- 구성(결론 1줄 +\
  \ 근거 3개)\n\n  - (1) 당사자/도메인 집착(진정성)\n\n  - (2) 테크/데이터로 제품화 가능한 역량(반복 실험, 시스템화)\n\
  \n  - (3) 운영/세일즈/CS 루프를 굴린 경험(트랙션으로 증명)\n\n### **4번(동기/목표) =** \n\n### **Antler에서\
  \ 무엇을 “증명”할지**\n\n- 목적: *Antler 자원을 써서 10주에 어떤 결과물을 만들 건지*\n\n- 구성(결론 1줄 + 목표 3개)\n\
  \n  - 목표는 “열심히” 말고 **증명할 지표/결과물**로\n\n  - 예: KPI 3개 검증, MVP 릴리즈, 가격/패키지 실험, 투자자\
  \ 스토리 정리\n\n---\n\n## **“두괄식 + 하단 구체” 방식, 이렇게 하면 딱 좋다**\n\n각 문항을 아래 포맷으로 통일해.\n\n\
  - **첫 줄: 결론(한 문장)**\n\n- **그 아래: 근거 3개(불릿)**\n\n- **마지막: 숫자는 딱 1번만(또는 4번만)**"
