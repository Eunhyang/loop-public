🎯 Near-Term Execution Plan (2026, 6–12개월)

LOOP의 2026 실행 계획은 “2027년 GLP-1 파동(Crash)을 타기 위한 준비”에 집중한다.

모든 행동(Task)은 Condition A(국내 PMF) 달성을 중심으로 정렬되어야 한다.

─────────────────────────────────

0. 원칙 (Principles)

- 연도 기반이 아니라 **Condition 기반**으로 움직인다.

• 2026년의 본질적 목표는 **2027년 GLP-1 Crash 파동을 탈 준비**를 완성하는 것이다.

• “가능한 것부터” — 준비된 단계만 Unlock한다.

• **유료화 = 데이터 품질 필터** (허수 제거 → 고밀도 사용자 확보)

• **코치 = Human-in-the-loop Labeler**

- 코칭은 서비스가 아니라 **데이터 라벨링 엔진**
    
    • 속도보다 방향 — OS Layer 구축 우선
    
    • Product–Data–Content–Coaching–Partnership–Runway 의 순서 고정

4. Track 간 의존성 맵 (Dependency)

Product → Data

• Loop Mapping 있어야 라벨링 데이터 증가

Data → Coaching

• 라벨링 툴 있어야 코치가 데이터 생산 가능

Content → Growth

• GLP-1 용어 선점 → 자연 유입 상승

Coaching → Data

• 고밀도 Tier 확보의 대다수는 코칭 기반

Data → Partnership

• 의료 제휴의 핵심은 Loop Dataset 품질
