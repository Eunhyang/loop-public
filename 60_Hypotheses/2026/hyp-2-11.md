# Exec↔Vault 통합 API가 실행→근거→결정 속도를 높인다

> Track: [[Track_2_Data]] | ID: `hyp-2-11` | 상태: planning

## 가설

**Q: Exec↔Vault 통합 API로 실행 기록이 자동 적재·조회 가능해지면 팀의 의사결정/반복 속도가 빨라지는가?**

---

## 판정 기준

### 성공 (Success)
도입 후 4주 내:
- (필수1) 실행 기록 생성→근거 조회까지의 중앙값이 **베이스라인 대비 ≥30% 단축**
- (필수2) 주간 결정 사이클(문제 정의→결정→실행 착수)이 **베이스라인 대비 ≥20% 단축**
- (필수3) 수기 적재/중복 입력 이슈 티켓이 **베이스라인 대비 ≥80% 감소**

### 전환 (Pivot)
도입 후 4주 내:
- 개선이 **<10%** 이거나
- 팀 내 주간 활성 사용자 비율이 **<30%** (실사용 정착 실패)
- 또는 데이터 정합성 문제(중복/누락)로 운영 비용/불신이 증가

---

## 측정 방법

**데이터 소스**: API 요청/이벤트 로그(적재·조회), Vault decision_log.jsonl, 프로젝트/태스크 업데이트 타임스탬프, (선택) PR/이슈 트래킹 타임라인

**측정 지표**:
- Time-to-Evidence(TTE): 실행 기록 생성 → 근거 조회까지 소요 시간(중앙값/분포)
- Decision Cycle Time(DCT): 문제 정의 → 결정 → 실행 착수까지 소요 시간(주간 단위)
- Manual Ops Rate: 수기 적재/중복 입력 관련 이슈 티켓 수(주간)
- Data Integrity: 중복/누락 탐지율(샘플링 QA 또는 자동 체크)
- Adoption: 주간 활성 사용자 비율(팀 내), 핵심 엔드포인트 사용률

## 전환 시나리오

### 영향
- “자동 적재·조회”만으로는 의사결정 속도 병목(정의/책임/회의/승인 등)이 해소되지 않았을 가능성
- 데이터 정합성/신뢰 문제가 생기면 오히려 ‘근거’로서 Vault 활용이 약화될 위험

### 대응 (피벗/중단)
- 피벗 1: API 자체보다 “결정 템플릿 + 자동 근거 첨부(링크/스냅샷)”까지 포함한 워크플로우로 확장
- 피벗 2: 통합 범위를 축소(핵심 엔드포인트 2~3개만)하고 정합성/UX를 우선 안정화
- 중단: 2회 반복 측정에도 개선 <10%이면, 기술 통합보다 운영 프로세스/책임 구조 개선을 우선 과제로 전환
