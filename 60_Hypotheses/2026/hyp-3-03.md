# 폭식 직전 트리거 푸시 개입 MVP - 콘텐츠 CTR 웨지 신호

> Track: [[Track_3_Content]] | ID: `hyp-3-03` | 상태: planning

## 가설

**Q: 꼭꼭 앱 잠재 사용자들이 평소 과식 트리거 상황(예: 금요일 18시 퇴근길 배달앱을 켤 때)에서 스트레스/습관으로 폭식 재발을 막기 어렵다면, 트리거 기반 푸시 알림으로 갈망 점수 체크와 10분 대체행동 제안을 노출했을 때 YouTube W03-26 콘텐츠에서 조회수 대비 앱 다운로드 링크 클릭 CTR이 3.0% 이상으로 증가하는가?**

### 관찰 기반 가정 (왜 이 가설을 하는가)

- **천천히 먹기 타이머 단일 기능**은 ‘확산/바이럴’ 웨지로는 약할 가능성이 높다.

  - 코칭 참여자 기준 4주 전후로 타이머가 **습관화되어 사용 빈도가 감소**하는 관찰이 있었다.

- 코칭에서 사람들이 가장 크게 “효과/변화”를 느끼는 지점은 기능 자체보다\
  **(1) 음식과의 관계 재해석**, **(2) 인정/자기비난 완화**, **(3) “아 내가 이 때 이렇게 자동으로 하는구나”라는 알아차림**에서 발생한다.

- 따라서 제품의 근본 웨지는 “일상 루틴 보조(타이머)”보다, 사용자가 가장 고통스러운 순간인 **폭식 직전/직후**에 개입하여\
  **패턴을 끊고(선택권 회복), 의미를 재해석해주는 경험**에 있을 가능성이 높다.

---

## 판정 기준 (성공/실패)

### 성공 (Success)

- 업로드 후 7일 이내
- 조회수 1,000 이상
- CTR(클릭/조회수) 3.0% 이상
- installs/1,000 views 10 이상

### 실패 (Failure)

- 업로드 후 7일 이내
- 조회수 1,000 이상
- CTR(클릭/조회수) 1.0% 미만 또는 installs/1,000 views 3 미만

## 측정 방법 (데이터 소스 + 지표)

데이터 소스

- YouTube: views
- Branch: campaign별 clicks, installs, click-to-install%

측정 지표

- Primary: CTR = clicks / views
- Guardrail: installs/1,000 views = installs / views x 1,000
- 참고: click-to-install% (Branch)

실험 방식

- W03-26 롱폼(및 필요 시 쇼츠 1\~2개)에 UI 오버레이로 1사이클 노출
- 오버레이 1사이클: 트리거 푸시 알림 → 갈망 점수 입력 → 대체행동 10분 제안/시작 → 10분 후 재점수 → 칭찬/다음 액션 1줄
- CTA는 영상당 1개 링크로 통일, 전용 Branch 캠페인 링크로만 유입

### 검증 철학 / 해석 규칙 (빠른 탐지기 실험)

- 이번 실험은 엄격한 A/B 테스트가 아니라, **‘강한 수요가 존재하면 잡음이 있어도 신호가 새어 나오는지**’를 확인하는 최소 검증이다.

- 즉, 조건 통제가 완벽하지 않더라도 **정말 게임체인저라면** 아래 중 최소 1개는 눈에 띄게 발생해야 한다:

  - **CTR/클릭 반응의 튐**

  - “링크/체크리스트/방법/앱 이름” 등 **요청형 댓글의 발생**

  - 저장/공유/DM 등 **비용 있는 행동 신호**

- 반대로 CTR이 낮고 요청형 댓글도 약하면, 이 컨셉은 ‘강한 웨지’ 후보가 아닐 가능성이 높다.\
  (단, 메시지/패키징/CTA/링크 동선의 영향은 존재하므로, 필요 시 1회는 포맷을 고정한 채 컨셉만 재시도한다.)
