# 하루 식사 계획(Upstream) MVP - 퇴근길 폭식 예방 웨지 신호

> Track: [[Track_3_Content]] | ID: `hyp-3-06` | 상태: planning

## 가설

**Q: 꼭꼭 앱 잠재 사용자들이 퇴근길(예: 평일 18~20시)에 무의식적으로 배달앱/편의점으로 과식·폭식 재발을 막기 어렵다면, 아침 또는 점심에 '오늘 만족 식사 1회'의 시간과 메뉴를 미리 정하고 리마인드+완료 체크를 제공했을 때 YouTube 콘텐츠에서 7일 내 링크 클릭 반응(CTR, 클릭수)이 baseline 대비 증가하는가?**

---

## 판정 기준

### 성공 (Success)
- 관찰 윈도우: 업로드 후 7일
- 표본 조건: views >= 2,000
- Reaction Signal(필수, 1개 이상 충족)
  - CTR(clicks/views) >= 1.5%
  - 또는 clicks/1,000 views >= 15
- 강한 신호(있으면 Validated에 준함)
  - 동일 포맷 baseline(일반 CTA 캠페인) 대비 clicks/1,000 views uplift >= 2.0배
  - 또는 요청형 댓글 3개 이상(예: "링크", "앱", "계획", "리마인드", "체크" 문의)
- 보조(Adoption Signal, 필수 아님)
  - opens/1,000 views >= 2
  - 또는 click->install >= 8%

### 실패 (Failure)
- 관찰 윈도우: 업로드 후 7일
- 표본 조건: views >= 2,000
- 아래를 모두 만족하면 Failure
  - CTR < 0.8%
  - 그리고 clicks/1,000 views < 8
  - 그리고 요청형 댓글 <= 1
- 클릭은 있으나 opens/installs가 거의 없고 uplift <= 1.2배이면 메시지/CTA/랜딩 마찰로 판단하고 동일 포맷으로 1회만 재시도 후 중단

---

## 측정 방법

**데이터 소스**: YouTube Analytics(views) + Branch Dashboard(Sources, Unique clicks/installs/opens, campaign별)

**실험 방식(운영 표준)**:
- 미드폼 또는 커뮤니티/쇼츠 중 1개 포맷을 선택해 1회 업로드/게시
- 영상/게시물 내에 Upstream(계획) 흐름을 1사이클로 시연
  - 아침/점심: 시간+메뉴 10초 설정
  - 해당 시간: 리마인드
  - 식사: 타이머 실행 또는 완료 체크
- CTA는 링크 1개로 통일하고, 전용 Branch 캠페인으로만 유입(예: landing2026w04, Unique ON)

**측정 지표**:
- Primary: CTR = clicks / views
- Primary(대체): clicks/1,000 views
- Secondary: opens/1,000 views, installs/1,000 views, click->install%
- Guardrail: 요청형 댓글 수(링크/앱/계획/리마인드/체크 문의)
